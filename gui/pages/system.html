MongoDB Key File

All Mongod instances in the cluster are deployed with a key file for Internal Authentication. Its contents can be copied from the text field below.

Administrative Account

All Mongod instances deployed by MAMID have user access control enabled. Replica Sets have an administrative user with the root role.

You may use this user for further configuration, e.g. adding additional users or configuring mongos instances.

Note: you must not change this credential manually, otherwise MAMID will lose management access to the Mongods it deployed.
